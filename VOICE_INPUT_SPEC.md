# macOS Voice Bridge - 语音输入桥接器需求文档

> 版本: v0.1  
> 日期: 2026-03-12  
> 状态: 需求阶段

---

## 1. 项目背景与目标

### 1.1 背景
macOS 系统自带语音听写功能交互体验良好，但存在以下问题：
- 中文识别准确率不高，特别是对技术术语、代码变量名、英文混合内容识别效果差
- 需要联网（部分情况）
- 无法本地优化词库

市面上缺乏一个**本地运行、高准确率、无 UI 干扰**的语音输入工具。

### 1.2 目标
开发一个轻量级语音输入桥接器，让用户通过快捷键即可调用高性能本地语音识别（Whisper），并将识别结果自动输入到当前光标位置。

**核心价值**:
- 保留 macOS 原生语音输入的交互体验，但替换为更准确的识别引擎
- **零侵入**: 不需要应用支持，模拟键盘输入，与目标应用解耦
- **本地运行**: 无需联网，保护隐私

## 2. 功能需求

### 2.1 核心功能

| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| **语音录制** | 按下 F5 开始录音，再次按下 F5 停止录音 | P0 |

| **语音识别** | 使用 Whisper.cpp 本地模型进行语音识别 | P0 |
| **自动输入** | 识别结果自动输入到当前聚焦的输入框 | P0 |
| **快捷键配置** | 支持自定义快捷键（默认 F5） | P1 |
| **模型选择** | 支持切换不同大小的 Whisper 模型 | P1 |
| **语言识别** | 自动检测中文/英文，或手动指定 | P1 |

**非功能**: 无 UI 界面，纯后台守护进程运行

### 2.2 交互流程

```
┌─────────────┐     按下 F5      ┌─────────────┐
│   待机状态   │ ──────────────▶ │   录音状态   │
│  (后台监听)  │                 │  (麦克风开)  │
└─────────────┘                 └─────────────┘
       ▲                                │
       │         再次按下 F5             │
       │   ◀────────────────────────────┘
       │
       │    ┌─────────────┐
       └─── │  语音识别中  │
            │ (后台处理)  │
            └──────┬──────┘
                   │
                   ▼
            ┌─────────────┐
            │  自动输入    │
            │ (模拟键盘)  │
            └─────────────┘

全程无 UI，仅通过系统蜂鸣声指示状态
```

### 2.3 状态说明

- **待机状态**: 后台守护进程运行，等待快捷键触发
- **录音状态**: 麦克风开启，音频数据写入内存/临时文件，可选声音提示（蜂鸣声）
- **处理状态**: 录音结束，调用 Whisper 进行识别（耗时 1-3 秒）
- **输入状态**: 模拟键盘事件，逐字输入识别结果

**状态指示方式**（无 GUI）:
- 录音开始: 播放短促提示音（或 Caps Lock LED 闪烁）
- 录音结束: 播放不同频率提示音
- 识别完成: 直接输入文字（用户通过输入框感知）

## 3. 非功能需求

### 3.1 性能要求
- 录音延迟: < 100ms（按下快捷键到开始录音）
- 识别延迟: 实时因子的 0.5x 以内（1 秒音频 0.5 秒内识别完成）
- 总延迟: 从停止录音到文字输入完成 < 3 秒（短句 < 1 秒）
- 输入延迟: 逐字输入间隔 10-20ms，模拟自然人打字

### 3.2 兼容性
- 操作系统: macOS 13.0+ (Intel & Apple Silicon)
- 目标应用: 任何支持键盘输入的应用程序
- 模型支持: Whisper.cpp ggml 格式模型

### 3.3 用户体验
- 无感安装: 提供一键安装脚本
- 后台运行: 随系统启动，无可见界面
- 失败处理: 识别失败时播放提示音，不干扰用户
- 隐私保护: 语音数据本地处理，不上传云端
- 静默运行: 除非出错，否则不打扰用户

## 4. 技术架构

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      User Layer                         │
│              (任意应用程序的输入框)                      │
└─────────────────────────────────────────────────────────┘
                           ▲
                           │ 模拟键盘输入
┌─────────────────────────────────────────────────────────┐
│                   Input Simulation                      │
│                    (CGEvent)                            │
│              macOS 底层键盘事件注入                      │
└─────────────────────────────────────────────────────────┘
                           ▲
                           │ 识别结果文本
┌─────────────────────────────────────────────────────────┐
│              Speech Recognition Engine                  │
│                   (Whisper.cpp)                         │
│            ┌──────────────────────┐                   │
│            │   ggml-medium.bin    │  (模型文件)        │
│            └──────────────────────┘                   │
└─────────────────────────────────────────────────────────┘
                           ▲
                           │ 音频文件路径
┌─────────────────────────────────────────────────────────┐
│                 Audio Recording                         │
│                 (AVFoundation)                          │
│         实时音频流  ──▶  PCM/WAV 文件                   │
└─────────────────────────────────────────────────────────┘
                           ▲
                           │ 全局快捷键事件
┌─────────────────────────────────────────────────────────┐
│                Hotkey Listener                          │
│                (NSEvent / CGEvent)                      │
│           后台全局监听 F5 按键                           │
└─────────────────────────────────────────────────────────┘
```

**设计原则**: 纯后台守护进程，无窗口、无菜单栏、无 Dock 图标

### 4.2 技术选型对比

| 组件 | 候选方案 | 推荐方案 | 理由 |
|------|----------|----------|------|
| 快捷键监听 | pynput / Carbon / NSEvent | **CGEventTap** | 底层全局监听，捕获 F5 无冲突 |
| 音频录制 | pyaudio / AVFoundation / SoX | **AVFoundation** | macOS 原生，权限管理完善 |
| 语音识别 | Whisper API / whisper.cpp | **whisper.cpp** | 本地运行，无网络延迟，隐私好 |
| 键盘模拟 | pyautogui / AppleScript / CGEvent | **CGEventPost** | 底层 API，兼容性好，可控制输入速度 |
| 后台常驻 | Python daemon / launchd | **launchd** | macOS 标准服务管理方式，无 UI |
| 状态指示 | 菜单栏 / 通知 / LED | **系统蜂鸣声** | 无 UI，最简单直接 |

### 4.3 推荐技术栈

- **语言**: Python 3.9+（开发效率）+ 必要的 C 扩展
- **核心依赖**:
  - `whisper-cpp-python`: Whisper.cpp 的 Python 绑定
  - `pyobjc-framework-CoreGraphics`: 调用 CGEvent API
  - `pyobjc-framework-AVFoundation`: 音频录制
  - `daemon`: Python 守护进程库

**无 UI 框架**: 不使用 rumps、tkinter 等任何 UI 库

## 5. 关键实现细节

### 5.1 状态机设计

```python
class VoiceInputState(Enum):
    IDLE = "idle"           # 待机
    RECORDING = "recording" # 录音中
    PROCESSING = "processing" # 识别中
    TYPING = "typing"       # 输入中

class VoiceInputController:
    def __init__(self):
        self.state = VoiceInputState.IDLE
        self.audio_buffer = []
        
    def on_hotkey(self):
        """F5 按键处理"""
        if self.state == VoiceInputState.IDLE:
            self.start_recording()
        elif self.state == VoiceInputState.RECORDING:
            self.stop_and_recognize()
        # 其他状态忽略或打断
```

### 5.2 音频处理流程

1. **录音阶段**:
   - 采样率: 16kHz（Whisper 推荐）
   - 格式: 16-bit PCM
   - 声道: 单声道
   - 存储: 内存缓冲区（短句）或临时文件（长句）

2. **预处理**:
   - VAD（语音活动检测）: 可选，用于自动截断静音
   - 归一化: 音量统一到标准范围

3. **识别参数**:
   - 语言: auto（自动检测）或 zh/en
   - 任务: transcribe（识别）
   - 模型: medium（平衡速度与精度）

### 5.3 键盘模拟策略

工具通过模拟键盘事件输入，需要兼容不同应用的输入行为：

- **方案 A - 直接粘贴**: 识别结果写入剪贴板，模拟 Cmd+V
  - 优点: 快，支持任意长度
  - 缺点: 覆盖用户剪贴板内容；某些应用（如 Vim）粘贴行为特殊

- **方案 B - 逐字输入**: 模拟键盘事件逐个字符输入
  - 优点: 不污染剪贴板，符合直觉，兼容所有应用
  - 缺点: 长文本较慢（100 字约 1-2 秒），需处理特殊字符（换行、Tab 等）

- **推荐**: 
  - 默认使用 **方案 B（逐字输入）**，保证最大兼容性
  - 提供配置项允许用户选择方案 A
  - 特殊字符处理: 中文直接输入，英文直接输入，换行符模拟 Enter 键

## 7. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 麦克风权限被拒绝 | 无法录音 | 首次启动检测，写入日志，静默退出 |
| 辅助功能权限不足 | 无法模拟键盘 | 首次使用时申请权限，日志提示 |
| Whisper 模型加载慢 | 首次识别延迟高 | 服务启动时预加载模型到内存 |
| 与其他快捷键冲突 | 功能异常 | 支持用户自定义快捷键，配置文件中修改 |
| 后台进程崩溃 | 服务不可用 | launchd 自动重启，日志记录 |
| 用户不知如何停止 | 困惑 | 提供卸载脚本和停止命令文档 |
| 在某些应用输入异常 | 兼容性 | 提供应用黑名单/白名单配置，或强制粘贴模式 |
| 游戏/全屏应用干扰 | 误触发 | 检测前台应用，支持排除特定应用 |
